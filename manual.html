📖 GAS Webhook セットアップガイド

AI Usage Tracker の GAS Webhook URL を取得するまでの手順

1Google Apps Script プロジェクトを作成

  1. Google Apps Script にアクセス
  2. 左上の 「＋ 新しいプロジェクト」 をクリック
  3. プロジェクト名を「AI Usage Tracker」などに変更（任意）

2コードを貼り付け

エディタに表示されている コード.gs のデフォルトコードを全て削除し、拡張機能に同梱されている gas/Code.gs の内容を貼り付けます。

PROJECT_ID を自身の GCPプロジェクトID に変更してください：

const PROJECT_ID = "your-gcp-project-id";  // ← ここを変更
const DATASET_ID = "ai_usage_logs";
const TABLE_ID   = "usage_events";
GCPプロジェクトIDは Google Cloud Console のプロジェクト選択画面で確認できます。

3BigQuery API サービスを有効化

  1. GASエディタ左サイドバーの 「サービス」 の横にある 「＋」 をクリック
  2. 一覧から 「BigQuery API」 を選択
  3. 識別子が BigQuery であることを確認して 「追加」 をクリック
GCP側でも BigQuery API が有効になっている必要があります。こちらから有効化できます。

4ウェブアプリとしてデプロイ

  1. GASエディタ右上の 「デプロイ」 → 「新しいデプロイ」 をクリック
  2. 種類の選択で 「ウェブアプリ」 を選択
  3. 以下の設定にしてください：
設定項目 値
説明 任意（例: "AI Usage Tracker v1"）
次のユーザーとして実行 自分
アクセスできるユーザー 全員
「次のユーザーとして実行」を「自分」、「アクセスできるユーザー」を「全員」に設定しないと、拡張機能からのリクエストが 401 エラーになります。

5Webhook URL をコピー

デプロイ完了後に表示される 「ウェブアプリ URL」 をコピーします。

https://script.google.com/macros/s/AKfycb.../exec

この URL を、拡張機能のオプション画面（設定画面）の 「GAS Webhook URL」 フィールドに貼り付けて保存してください。

6接続テスト

オプション画面の 「💾 保存して接続テスト」 ボタンを押すと、自動的にテストリクエストを送信します。

「✅ 接続成功」と表示されればセットアップ完了です。

BigQueryのデータセット・テーブルが未作成の場合は、最初のデータ送信時に自動的に作成されます。

📊 BigQuery テーブルスキーマ

以下のスキーマで ai_usage_logs.usage_events テーブルが自動作成されます：

カラム名 型 説明
user_email STRING ユーザーメールアドレス
service_name STRING AIサービス名
action STRING アクション種別
timestamp TIMESTAMP 検知日時
url STRING 利用URL
inserted_at TIMESTAMP BQ挿入日時

🔄 GASコードの更新方法

GASのコードを修正した場合は、再デプロイが必要です：

  1. 「デプロイ」→「デプロイを管理」
  2. 鉛筆（✏️）アイコンをクリック
  3. バージョンを 「新しいバージョン」 に変更
  4. 「デプロイ」をクリック
バージョンを「新しいバージョン」に変更しないと、古いコードのまま実行されます。
